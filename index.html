청주 교통사고 위험도 분석 & LLM 요약 서비스

개인 프로젝트

1. 어떤 걸 만든 건지
청주시 교통사고 데이터를 기반으로 구·동별 위험도 점수를 계산하고,
LLM이 각 지역의 주요 위험 요인과 우선 대응 전략 3가지를 요약해 주는 웹 서비스입니다.

2. 왜 만들었는지
기존 교통사고 분석은 단순 사고 건수/사망자 수 지도로 끝나는 경우가 많습니다.
이 프로젝트에서는 “행정 담당자가 바로 정책 아이디어를 얻을 수 있는 분석”을 목표로, 위험도 점수와 함께 LLM이 문장으로 전략을 제안하도록 차별화를 두었습니다.

3. 내가 맡은 역할
· 데이터 분석 & 백엔드 & 간단한 프론트엔드 전체를 혼자 구현
· 교통사고 데이터 전처리, 위험도 산식 정의, Streamlit UI 구성
· LLM 프롬프트 설계 및 요약 결과 튜닝

4. 인프라 구성
· 데이터 분석: Jupyter Notebook + Pandas + SQLite
· 웹 앱: Streamlit
· 배포 & 버전 관리: GitHub 레포지토리 기반(로컬 실행 + 배포 확장 전제)

2025.08 – 2025.09

네이버 뉴스 댓글 감성 분석

개인 프로젝트

1. 어떤 걸 만든 건지
네이버 뉴스 댓글을 수집해 이슈·기간별 긍/부정 비율과 키워드 분포를 분석하고,
대시보드로 시각화한 프로젝트입니다.

2. 왜 만들었는지
이미 뉴스 요약 서비스는 많지만, “사람들이 어떻게 반응했는지”를 보여주는 서비스는 상대적으로 적습니다.
이 프로젝트는 기사 내용보다 댓글의 감정 흐름과 키워드 변화에 초점을 맞춰 커뮤니티 분위기를 한눈에 볼 수 있도록 차별화했습니다.

3. 내가 맡은 역할
· 크롤러 설계 및 구현 (Selenium + BeautifulSoup)
· 형태소 분석/감성 사전 기반 감성 점수 계산 파이프라인 구축
· Power BI를 활용한 시각화 대시보드 설계

4. 인프라 구성
· 데이터 수집: Python + Selenium 크롤링 스크립트
· 저장: CSV & SQLite 혼합 사용
· 분석/시각화: Jupyter Notebook → Power BI 연결
· 코드 관리: GitHub 레포지토리

2025.07 – 2025.08

주문 데이터 자동 수집 & 매출 리포트 RPA

개인 프로젝트

1. 어떤 걸 만든 건지
쇼핑몰 주문 데이터를 주기적으로 자동 수집해서
일·주간 매출 리포트를 생성하고, 슬랙으로 결과를 보내는 RPA 파이프라인입니다.

2. 왜 만들었는지
실제 운영에서는 담당자가 매일 엑셀을 내려받고 피벗 테이블을 만드는 반복 업무가 많습니다.
이 프로젝트는 그런 반복 작업을 완전히 자동화하는 엔드투엔드 파이프라인을 만드는 데 초점을 두어, “한 번 설정해두면 계속 돌아가는” 구조를 목표로 했습니다.

3. 내가 맡은 역할
· 전체 파이프라인 구조 설계
· 주문 데이터 크롤링/수집 스크립트 구현
· 매출 집계 로직 및 리포트 포맷 설계
· Slack Webhook 연동으로 알림 자동화

4. 인프라 구성
· 수집/집계 스크립트: Python
· 스케줄링: OS 작업 스케줄러 기준으로 설계 (crontab/작업 스케줄러)
· 저장소: CSV + SQLite
· 알림: Slack Webhook
· 코드 및 설정: GitHub 레포지토리 관리

AI 기반 스트레스 분석 앱

사이드 프로젝트

1. 어떤 걸 만든 건지
사용자가 텍스트로 하루를 기록하면, LLM이 내용을 분석해
스트레스 수준을 점수화하고 간단한 행동 가이드를 제안해 주는 앱입니다.

2. 왜 만들었는지
단순 일기 앱은 많지만, “내 상태를 숫자와 문장으로 피드백해 주는 서비스”는 상대적으로 적습니다.
이 프로젝트는 기록 자체보다 기록 이후의 피드백에 집중해서, 사용자가 자신의 스트레스 패턴을 이해하는 데 도움을 주고자 했습니다.

3. 내가 맡은 역할
· 백엔드 API 설계 및 구현 (FastAPI)
· LLM 기반 감정/스트레스 분석 파이프라인 설계
· JWT 기반 인증 및 세션 구조 설계

4. 인프라 구성
· 백엔드: FastAPI (Python)
· 인증: JWT + 간단한 사용자 관리
· 모델 연동: OpenAI API 호출 레이어
· 배포/코드 관리: GitHub + (추후 클라우드 배포를 전제로 설계)

3. 내가 맡은 역할
· 더미데이터 스키마 설계 및 생성
