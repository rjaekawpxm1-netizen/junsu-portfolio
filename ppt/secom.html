SECOM 반도체 불량 예측

개인 프로젝트 · 분석·모델링

분석·모델링
GitHub Repository ↗

1프로젝트 개요

센서가 590개 넘는 반도체 공정 데이터에서 불량을 예측한 프로젝트입니다.
어느 센서가 불량과 관련 있는지도 함께 찾았습니다.
데이터가 많고 결측이 많고 불량이 드문 상황에서, 쓸 만한 예측을 내는 게 목표였습니다.

담당 역할

전처리, 변수 선택, 모델링, 해석까지 혼자 했습니다.

2문제 정의 · 추진 배경

반도체 공정은 센서가 수백 개고 불량은 드물게 납니다.
그래서 어느 구간이 불량과 관련 있는지 짚기가 어렵습니다.
이런 데이터에서도 불량을 실제로 잡아낼 수 있을지 해보고 싶었습니다.

기존 방식의 한계

  • 센서가 590개 넘게 많고 결측이 많다.
  • 불량이 6.6%쯤이라 한쪽으로 치우쳐 있다.
  • 선형 모델로는 복잡한 패턴을 잡기 어렵다.
핵심 질문

“센서 많고 결측 많고 불량 드문 데이터에서 불량 구간을 실제로 짚어낼 수 있을까?”

3데이터 · 입력 정보

  • SECOM 공정 센서 데이터(590개 이상)
  • 불량/정상 라벨(불량 6.6%쯤)

결측 많고 쓸모 적은 변수를 빼서 수를 줄였습니다.
치우침을 보정한 다음, 복잡한 패턴도 잡는 모델로 학습했습니다.

4해결 접근 방식

  • 결측 많고(70%↑) 변화 적고 서로 겹치는 변수를 빼서 590개 이상 → 202개로 줄였다.
  • Logistic Regression(AUC 0.68)보다 나은 XGBoost로 학습했다.
  • 기준값을 0.3으로 낮춰 불량을 더 잡고, 중요도·SHAP로 주요 센서를 봤다.

5문제 해결 과정

문제 1. 데이터가 많고 결측이 많음
문제 상황센서 590개 이상과 많은 결측이 학습을 방해했습니다.
해결 과정
  • 결측 많고 변화 적고 겹치는 변수를 뺐다.
  • 590개 이상 → 202개로 줄였다.
결과잡음이 줄어 학습이 더 안정적이었다.
문제 2. 불량이 드물어 잘 못 잡음
문제 상황불량이 6.6%쯤이라 모델이 불량을 잘 못 골랐습니다.
해결 과정
  • 치우침을 보정했다(scale_pos_weight).
  • 기준값을 0.3으로 낮춰 불량을 더 잡았다.
결과불량을 더 많이 찾아 실무에 쓸 만해졌다.
문제 3. 선형 모델의 한계
문제 상황Logistic Regression은 AUC 0.68로 패턴을 충분히 못 잡았습니다.
해결 과정
  • XGBoost로 복잡한 패턴까지 학습했다.
  • 중요도·SHAP로 주요 센서를 해석했다.
결과ROC-AUC 0.80쯤, 불량 영향 상위 20개 센서를 찾았다.

6프로젝트 의의

  • 많고 치우친 공정 데이터를 직접 전처리·모델링해봤다.
  • 선형에서 복잡한 모델로 바꾸고 기준값을 조정해 실무에 맞춰봤다.
  • SHAP로 모델을 해석해 점검 시나리오로 이어봤다.
