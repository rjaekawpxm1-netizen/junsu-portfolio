성평등가족부 AI 편향 탐지 시스템

1프로젝트 개요

공개 데이터와 LLM에 성별 편향이 있는지 여러 방법으로 재보는 도구입니다.
문장을 바꿔 비교하기, WEAT, 일부러 까다로운 질문 던지기 등을 써서 편향을 숫자로 볼 수 있게 만들었습니다.

담당 역할

편향을 재는 방법을 정하고 구현했고, 모델 비교와 대시보드까지 혼자 만들었습니다.

2문제 정의 · 추진 배경

AI 데이터나 모델에 성별 편향이 있을 수 있다는 걱정은 많습니다.
그런데 정작 내보내기 전에 직접 확인할 도구는 없었습니다.
그래서 편향을 숫자로 재서 미리 점검할 수 있을지 해봤습니다.

기존 방식의 한계

  • 편향을 직접 점검할 객관적 지표가 없다.
  • 지표 하나만으로는 편향이 잘 안 드러난다.
  • 모델마다 편향이 달라 비교 기준이 필요하다.
핵심 질문

“AI 데이터와 모델의 성별 편향을 숫자로 재서, 내보내기 전에 점검할 수 있을까?”

3데이터 · 입력 정보

  • 뜻은 같고 성별만 바꾼 문장쌍 약 100건
  • WEAT 측정용 단어셋
  • 까다로운 질문(적대적 프롬프트) 세트
  • 분석 데이터 345개 · 실데이터 84,229건

여러 방법을 같이 써서 한 지표의 한계를 메웠습니다.
같은 입력을 Claude와 GPT-4o에 넣어 결과를 비교했습니다.

4해결 접근 방식

  • 성별만 바꾼 문장 비교, WEAT, 까다로운 질문, 복합 편향 등 여러 방법을 함께 썼다.
  • Claude와 GPT-4o 결과를 나란히 놓고 모델별 차이를 봤다.
  • 데이터 익명화·품질 점검, 벤치마크셋·지시 데이터 만들기까지 정리했다.

5문제 해결 과정

문제 1. 점검할 도구가 없었음
문제 상황데이터·모델의 편향을 내보내기 전에 잴 방법이 없었습니다.
해결 과정
  • 문장 비교·WEAT·까다로운 질문을 한 도구로 묶었다.
  • 결과를 CSV로 남겨 다시 돌려볼 수 있게 했다.
결과내보내기 전에 직접 점검할 수 있는 도구를 갖췄다.
문제 2. 지표 하나로는 부족함
문제 상황한 가지 지표만 보면 놓치는 편향이 있었습니다.
해결 과정
  • 여러 방법을 같이 적용했다.
  • 복합 편향까지 함께 봤다.
결과한 지표가 놓치는 편향을 여러 방법으로 잡았다.
문제 3. 모델별 차이를 비교해야 함
문제 상황어느 모델이 더 편향됐는지 비교할 기준이 없었습니다.
해결 과정
  • Claude와 GPT-4o 결과를 나란히 비교했다.
  • Streamlit 대시보드로 한눈에 봤다.
결과모델별 편향 차이를 숫자로 비교했다.

6프로젝트 의의

  • 편향은 한 지표로 안 잡힌다고 보고 여러 방법을 겹쳐서 설계했다.
  • 모델을 비교해 편향 차이를 숫자로 봤다.
  • 데이터·벤치마크셋 만들기까지 이어봤다.
