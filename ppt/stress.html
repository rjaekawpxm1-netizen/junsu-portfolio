1프로젝트 개요

얼굴 표정, 음성, 설문을 함께 써서 스트레스 상태를 분석하는 앱입니다.
한 가지 데이터만 보지 않고 셋을 묶어서 더 안정적으로 판단하게 만들었습니다.
분석한 결과에 맞춰 음악과 영화도 추천해 줍니다.

담당 역할

팀 프로젝트였고, 저는 팀장을 맡았습니다.
프로젝트 방향을 정하고 데이터 전처리와 분석 구조 설계를 주로 했습니다.

2문제 정의 · 추진 배경

설문으로 스트레스를 재면 그날 기분에 휘둘리고, 꾸준히 챙기기도 어렵습니다.
웨어러블은 기기를 따로 사야 하고요.
그래서 평소 쓰는 휴대폰만으로 스트레스를 볼 수 있을지 궁금했습니다.

기존 방식의 한계

  • 설문은 주관적이라 그때그때 다르고 계속 관리하기 어렵다.
  • 웨어러블은 돈이 들고 기기에 의존하게 된다.
  • 데이터 하나만 쓰면 결과가 잘 안 맞는다.
핵심 질문

“평소 쓰는 휴대폰만으로 스트레스 상태를 자동으로 볼 수 있을까?”

3데이터 · 입력 정보

  • 얼굴 표정 이미지
  • 음성 감정 데이터
  • 설문 응답

AIHub의 감정 데이터를 썼습니다.
세 가지를 따로 분석한 다음, 그 결과를 합쳐 최종 점수를 냈습니다.

4해결 접근 방식

  • 얼굴·음성·설문을 각각 분석하고, 그 결과를 합치는 구조로 만들었다.
  • 세 결과를 가중 평균해서 0~100 점수로 바꿨다.
  • 점수에 맞춰 음악·영화를 추천해, 분석이 화면에서 끝나지 않게 했다.

5문제 해결 과정

문제 1. 데이터 하나만으로는 잘 안 맞음
문제 상황얼굴이나 음성 하나만 보면 감정이 틀리게 잡힐 때가 있었습니다.
해결 과정
  • 얼굴·음성·설문을 같이 보도록 바꿨다.
  • 세 결과를 가중 평균으로 합쳤다.
결과하나만 쓸 때보다 결과가 더 안정적이었다.
문제 2. 결과를 어디에 쓸지 막막함
문제 상황분석 결과가 숫자로만 나와서 정작 쓸 데가 없었습니다.
해결 과정
  • 감정에 맞는 음악·영화 추천을 붙였다.
  • 분석 결과를 추천과 이었다.
결과분석한 내용이 실제로 써먹는 기능이 되었다.
문제 3. 실제 서비스로 굴릴 구조가 필요함
문제 상황모델은 있는데 앱으로 어떻게 돌릴지가 안 잡혀 있었습니다.
해결 과정
  • 앱·서버·AI 분석 서버를 나눠서 설계했다.
  • 사용자 기록을 저장하는 구조를 만들었다.
결과실제 서비스로 옮길 수 있는 형태를 갖췄다.

6프로젝트 의의

  • 여러 데이터를 묶어 스트레스를 분석하는 구조를 직접 짜봤다.
  • 분석 결과를 추천 기능까지 연결해봤다.
  • 팀장으로 사람들과 같이 만들어봤다.
