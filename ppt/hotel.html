호텔 예약 위험도 분석

팀 프로젝트 · 팀장

팀장
GitHub Repository ↗

1프로젝트 개요

호텔 예약 데이터를 보면서, 기준을 "취소율"에서 "취소되면 손실이 얼마인지"로 바꿨습니다.
그렇게 보니 정말 챙겨야 할 위험한 예약이 따로 보였습니다.
어떤 기준을 보느냐에 따라 결론이 달라진다는 걸 보여주고 싶었습니다.

담당 역할

팀 프로젝트였고, 저는 팀장으로 문제를 다시 정의하고 분석 방향을 잡았습니다.

2문제 정의 · 추진 배경

취소율이 높다고 꼭 손실이 큰 건 아니었습니다.
그래서 취소율 대신 손실 규모로 봐야 하지 않을까 싶어 기준을 바꿔봤습니다.

기존 방식의 한계

  • 취소율 높은 예약이 꼭 큰 손실은 아니다.
  • 환불·채널·보증금에 대한 위험 기준이 없다.
  • 조건 하나만 보면 진짜 위험을 놓친다.
핵심 질문

“취소율 대신 손실 규모로 위험을 보면, 무엇을 챙겨야 할까?”

3데이터 · 입력 정보

  • 호텔 예약 데이터(취소 여부·환불·채널·보증금 등)
  • 예약별 손실 금액

취소 여부가 아니라, 취소되면 손실이 얼마인지를 기준으로 삼았습니다.
환불·채널·보증금 조합별로 손실을 비교했습니다.

4해결 접근 방식

  • 기준을 취소 여부에서 손실 금액으로 바꿨다.
  • 환불 여부 × 채널 × 보증금 조합별로 손실을 비교했다.
  • 취소율은 낮아도 손실 큰 예약을 위험 조합으로 잡았다.

5문제 해결 과정

문제 1. 취소율 기준의 함정
문제 상황취소율이 높은 예약이 꼭 큰 손실은 아니라서 우선순위가 어긋났습니다.
해결 과정
  • 기준을 취소 여부에서 손실 금액으로 바꿨다.
  • 총·평균 손실로 예약을 다시 봤다.
결과정말 챙겨야 할 위험 예약에 집중하게 됐다.
문제 2. 조건 하나로는 안 보임
문제 상황환불·채널·보증금을 따로 보면 위험이 잘 안 드러났습니다.
해결 과정
  • 환불×채널×보증금 조합별로 손실을 비교했다.
  • 조합별 위험 구간을 찾았다.
결과조건이 아니라 조합이 위험을 만든다는 걸 확인했다.
문제 3. 분석을 결정으로 이어야 함
문제 상황분석이 숫자로 끝나면 정책 개선으로 안 이어집니다.
해결 과정
  • 위험 조합을 바탕으로 보증금·채널 정책 개선안을 냈다.
결과취소율 기준을 손실 기준으로 바꿨다.

6프로젝트 의의

  • 기준을 바꾸면 결론이 달라진다는 걸 직접 보여줬다.
  • 조합으로 봐서 숨은 위험 구간을 찾았다.
  • 분석을 정책 제안까지 이어봤다.
